# Aurora Terminal Plugin API Reference

Aurora plugins run on the Extism Wasm engine. Plugins respond to the host via **Exported Functions** and access system capabilities via **Host Functions**.

## 1. Exported Functions (Implemented by Developer)

These functions are defined by the plugin and called by Aurora Terminal during specific lifecycle stages.

### `connect(settings_json: string) -> string`
- **Trigger**: When the user clicks the "Connect" button in the UI.
- **Parameters**: `settings_json` (String) - A JSON string containing Host, Port, Auth info, etc.
- **Returns**: `connection_id` (String) - A unique identifier for the connection generated by the plugin.

### `send(connection_id: string, data: Vec<u8>)`
- **Trigger**: When the user types and sends data in the terminal.
- **Action**: The plugin should send the byte stream through its established socket.

### `process(data: Vec<u8>) -> Vec<u8>`
- **Trigger**: Core interceptor. Called when data flows through the terminal.
- **Action**: Used for protocol parsing, decryption, or data extraction. Returns processed data for display or transmission.

### `disconnect(connection_id: string)`
- **Trigger**: When the user disconnects or closes the tab.
- **Action**: The plugin should clean up and close the corresponding socket connection.

---

## 2. Host Functions (Provided by Aurora)

These functions are injected into the Wasm sandbox by Aurora Terminal and can be called by the plugin.

### `emit_data(connection_id: string, data: Vec<u8>)`
- **Description**: **Critical**. Since network reception is usually asynchronous, the plugin must call this function to push received data back to the terminal UI.

### `host_exec(command: string) -> string`
- **Description**: Allows the plugin to execute restricted shell commands on the host system (e.g., `git status`).

### `host_log(level: string, message: string)`
- **Description**: Outputs plugin logs to the Aurora Terminal developer console. `level` can be: `info`, `warn`, `error`.
